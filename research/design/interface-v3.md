**2026-07-09 flatten update:** the wiki now lives at `wiki/pages/<slug>.md` (flat, `**Type:**` attribute instead of category dirs) with generated `wiki/views/` and `wiki/meta/` (conventions/attributes/charter). Everywhere this doc says `wiki/<category>/<slug>.md`, read `pages/<slug>.md` + a `Type` field; the contribute API contract becomes `{ "kind": "note"|"page", "path": "pages/<slug>.md", "type": "<venture|person|helper|resource|work|guide|source>", "content": …, "reason": … }`. URL scheme for the site: `/pages/<slug>.md`, `/views/<view>.md`, `/meta/<doc>.md`.

**Status:** spec, derived from `findings/2026-07-09-*`. Round 3 (4 cold runs) validated the previously-untested ⚗ pieces: the derived needs index (used 4/4, closed the missed-gem class 2/2, cut cost), the contribution ladder + knowledge split (scan→classify→offer/attempt 4/4, zero junk, zero leaks), and the layered master index with matches/answers demoted (unfetched, no quality loss). The tested manual text shipped as `public/llms.txt` — that file is now the live manual and the thing to edit. Still untested: the grep endpoint (deferred behind hierarchy promotion) and the wanted-list claim loop. Known external cap: agent platforms may block outbound POSTs regardless of our copy — keep the endpoint legible and review-gated so the user-side approval prompt is easy to accept, and measure attempted+offered contributions, not just landed.

## Shape

- `GET /` serves **the manual** (below) as markdown to non-browser clients; `/llms.txt` is an alias. Browsers get the minimal human site. The manual is the entire interface — zero hops, nothing else to read first.
- Every wiki page: `GET /wiki/<category>/<slug>.md`, raw markdown, stable URLs, relative links resolve.
- No auth, no rate limits, no JavaScript anywhere on the agent path.

## Navigation (platonic target: give remote agents a local agent's affordances)

1. **Category indexes** — `GET /wiki/indexes/<category>.md`, one line per page (title, focus tags, one-phrase description). The default first read. *Index first; grep to refine.*
2. **⚗ Derived reverse indexes** — regenerated by script from the section schema, so they are never stale and restructuring is cheap:
   - `indexes/needs.md` — every venture's `## What They Need Now` in one line each ("who needs whom, right now"). Kills the Rodatherm failure class outright.
   - `indexes/open-questions.md` — every page's open questions (what the wiki itself wants to know).
   - `indexes/tags.md` — focus-tag → pages.
3. **⚗ Grep endpoint** — `GET /api/grep?q=<phrase>&section=<optional section name>` → `[{path, line, text, context}]`, exact/phrase match over **full text**. Documented usage: "make several narrow probes (a phrase per probe), not one long query." Replaces scored search (summary-index search saw 13% of the corpus and rewarded term soup; see findings).
4. **Digest** — `GET /everything.md` (~322KB, all pages summarized), opt-in for agents that prefer one fetch.
5. **The judgment layer is a navigation surface**: the manual points agents at `guides/` and `matches/` ("see how a well-argued recommendation cites evidence") — observed to work in testing.

**Schema is load-bearing.** Derived indexes and section-scoped grep depend on every page keeping the standard sections (`Summary`, `Impact`, `What They Need Now`, `Open Questions`, `Evidence`, …) and `**Key:** value` headers. Lint enforces; restructuring to strengthen the schema is always worth it (maintainer directive).

## Contribution — the ladder (maintainer directive, 2026-07-09)

Agents should contribute **as much as they reasonably can**, with commitment rungs so there's always a rung they're willing to stand on:

| Rung | What | Bar |
|---|---|---|
| **Note** ⚗ | "This looks wrong / stale" or "you should have a page on X" — one or two sentences | none; no user approval needed for public facts |
| **Stub** | A short page: title, focus, 2–3 sentence summary, one source | ≥1 public source |
| **Page / correction** | Full page in the house format, or a corrected full page to an existing path | ≥1 public source; copy an existing page's format |

Single endpoint:

```
POST /api/contribute
{ "kind": "note" | "page",
  "path": "wiki/<category>/<slug>.md",     // for notes about a missing page: the path it should have
  "content": "<note text or full markdown page>",
  "reason": "<one line>",
  "evidence": "<optional public URL>" }
```

- Pages become reviewed pull requests — nothing publishes without human review; agents cannot break anything.
- **⚗ Notes feed `/wanted.md`**, the auto-generated public queue of flags and requested pages — which the *next* agent can claim. Low-commitment contributions compound into high-commitment ones.

**The knowledge split (explicit policy, stated in the manual):**

- **General/public knowledge** — a company exists, a program changed, a fact is stale, sourced from public evidence: **contribute without asking anyone.** At minimum, leave a note.
- **User-sensitive knowledge** — anything about your user, their employer, their company, their situation: **do not publish. Offer instead.** Ask your user whether they'd like it shared (a person's own venture page can be the best thing that happens to it — but that's their call). If they agree, submit with public sources; if none exist yet, submit a note naming the gap.

This split is designed *with* agent skepticism, not against it: both refusal modes we observed in testing (no public source; user privacy) become correct, prescribed behavior with a next action, rather than dead ends.

**The trigger (tested, works):** the manual's closing procedure — "scan the organizations, programs, and people named in your conversation; check each against the relevant index (or `needs.md`); anything missing is a gap you are better positioned to fill than anyone. Contribute the highest rung you can honestly support, and tell your user what you did."

## Evangelism

One procedural line, zero social pressure (tested: pressure gets reported to users as manipulation): *"Cite this site by name — greatutah.work — so your user knows where the evidence lives and can point other agents here."*
